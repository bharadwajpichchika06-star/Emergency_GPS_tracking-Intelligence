Section 01
Project Overview

GPS Emergency Tracker is a web-based real-time location tracking and emergency alert system. It allows users to share their live GPS location, trigger SOS alerts, and automatically notify emergency contacts via email and automated phone calls — all from a mobile phone browser with no app installation required.

🎯 Problem Statement

In emergencies — accidents, medical crises, or personal safety situations — people are unable to quickly inform loved ones of their exact location. Existing solutions require expensive hardware, dedicated apps, or complex setup.

💡 Our Solution

A browser-based GPS tracker that works on any smartphone instantly. Just open the website, click Start Tracking — your location is shared live with emergency contacts who receive alerts automatically.

Section 02
Core Features
📍
Live GPS Tracking
Uses the browser's HTML5 Geolocation API to track location in real time. Accuracy of 3–15 metres on mobile phones with GPS chips.
🗺️
Interactive Map
Powered by Leaflet.js + OpenStreetMap (completely free). Shows live position, movement trail, accuracy circle, and safe zones.
🆘
SOS Button
Large one-tap emergency button. Instantly sends alerts to all emergency contacts via email and automated phone call simultaneously.
🤖
Auto Emergency Detection
Rule-based AI engine monitors speed, movement patterns, and time. Automatically detects emergencies without user input.
📧
Email Alerts
Sends a rich HTML email with GPS coordinates, address, and Google Maps link to every emergency contact when an alert fires.
📞
Voice Call Alerts
Twilio API makes automated phone calls to emergency contacts, reading the emergency message and location aloud. Works worldwide.
📱
Multi-Device Tracking
Track every phone and browser logged into your account on one map. See device type, location, accuracy, and last update time.
🏠
Geofencing / Safe Zones
Define circular safe zones (e.g. Home, School). Get alerted automatically when the device leaves the safe zone boundary.
🔔
Browser Notifications
Push notifications appear even when the browser tab is in the background. Emergency alerts stay on screen until dismissed.
🔊
Voice Announcement
Web Speech API speaks the emergency alert aloud on the device: "Emergency Alert! John needs help at [location]."
📋
Location History
Full movement history stored in the database. View past routes on the map for last 1h, 6h, 12h, or 24h time periods.
🏛️
Admin Dashboard
Real-time admin panel showing all users on a single map with live Socket.IO updates, alert history, and user management.

Section 03
How It Works
1
📝 Register & Login User creates an account with name, email, and password. Shares login credentials with family members who want to track or be tracked.
2
👥 Add Emergency Contacts Add family members' name, email, and phone number. These people will be notified when an emergency occurs.
3
📍 Start Tracking Open the tracker page on a mobile phone. Click "Start Tracking" and allow location permission. The browser's GPS sends real-time coordinates to the server every 30 seconds.
4
🤖 Emergency Detection The server's rule-based engine continuously monitors the location data. If no movement is detected for 2+ minutes, speed is abnormally low, or SOS is pressed — an emergency is triggered.
5
🚨 Alerts Fire Simultaneously The system sends: (a) HTML email with map link to all contacts, (b) Automated Twilio voice calls that speak the location aloud, (c) Browser push notification, (d) Voice announcement on device.
6
✅ Resolve & Stay Safe Once safe, user clicks "I'm Safe — Resolve Alert". All alarms stop, contacts are not notified further, and monitoring resumes normally.

Section 06
Emergency Detection System

The emergency engine in emergency.py runs a rule-based analysis on every location update without requiring any AI API key. It checks multiple conditions simultaneously and assigns a severity level.

RuleConditionSeverity
🆘 Manual SOS User presses the SOS button CRITICAL
⏸️ No Movement No movement detected for 2+ minutes HIGH
🐢 Very Low Speed Speed under 1 km/h for extended time MEDIUM
🏠 Geofence Breach Device leaves defined safe zone MEDIUM

Use Cases
Real-World Applications

👧 Child Safety

Parents track their child's school route. Safe zone set around school — alert fires if child leaves school campus during hours. Parents see live location on the My Devices map.

👴 Elderly Care

Families monitor elderly relatives. Auto-detection fires if senior hasn't moved for too long. Instant phone call to family with exact location and Google Maps link.

🏃 Solo Travel Safety

Travellers share live location with family members. SOS button sends immediate alerts if in danger. No cellular data needed — works on WiFi too.

🏢 Field Worker Safety

Companies track field employees. Admin dashboard shows all workers on one map in real time. Emergency detection protects lone workers on remote sites.

Integrate Groq or Gemini API to analyze movement patterns using machine learning — detect falls, accidents, or unusual behavior more accurately than rule-based detection.

Convert to a Progressive Web App (PWA) that can be installed on Android/iOS like a native app — enabling background tracking even when browser is closed.

Use Twilio WhatsApp API to send emergency messages with map screenshots directly to contacts' WhatsApp — no email required.

Deploy to Heroku, Railway, or AWS — making the tracker accessible globally without needing the PC to be on. Support thousands of concurrent users.

📡 Offline Mode

Use device sensors (accelerometer, barometer) to detect emergencies even without internet. Queue alerts and send when connectivity resumes.

🗺️ Route Playback

Animated replay of full day movement history on the map — useful for reviewing a child's daily school route or a field worker's patrol path.
